# CLAUDE.md

开始任何分析、设计、编码、重构之前，先读：

1. `docs/ARCHITECTURE.md`
2. `docs/UI_SPEC.md`

如果实现与文档冲突，以文档为准。

## 工作原则

- 先按文档实现，不要重新发明方案
- 不擅自引入文档之外的新框架、新模块、新抽象层
- 优先最小闭环，先做可运行、可验证的实现
- 每次改动只收敛在当前任务相关文件，避免无关重构
- 预览与导出必须保持同一套组件和样式
- 如果文档没写清楚，采用最保守、最简单、最容易维护的实现，不要自行扩展

## 实现要求

- 代码必须可运行，不能留伪代码或空实现
- 输入输出要有明确类型
- 能复用共享类型时不要重复定义
- 路由层保持轻量，业务逻辑与数据访问分开
- 所有输入都要校验
- UI 不要偏离 `docs/UI_SPEC.md`

## 完成标准

功能完成不仅是代码写完，还要满足：

- 本地可运行
- 主流程可验证
- 与文档一致
- 不引入无关复杂度
