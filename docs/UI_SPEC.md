# UI_SPEC.md

## 目标

UI 风格定为：**浅色、专业、克制、结构化的桌面工具台**。  
不要做成官网、作品集、营销页或炫技界面。

## 布局

主页面固定三栏：

- 左栏：输入与操作，宽 `320px`
- 中栏：结构化编辑，宽 `420px`
- 右栏：实时预览，自适应

另有导出页：

- `/export/:jobId`

不改三栏结构，不做复杂导航，不做花哨动效。

## 视觉

- 页面背景：`#F8FAFC`
- 面板背景：`#FFFFFF`
- 主文字：`#0F172A`
- 次级文字：`#475569`
- 主色：`#2563EB`
- 边框：`#E2E8F0`

字体只用系统字体栈，不引入外部字体。

## 风格约束

- 留白充分，信息清晰，强调“编辑”和“预览”
- 右侧预览是主视觉区域，但仍然是工具预览，不是展示海报页
- 预览与导出必须复用同一套卡片组件和样式
- 第一版只允许一套浅色主题

## 组件风格

- 按钮、输入框、面板风格统一
- 圆角和阴影保持克制
- 图标统一线性风格
- 不混用多套 UI 语言

## 禁止项

不要使用：

- 玻璃拟态
- 大面积渐变
- 彩色阴影
- 发光效果
- 花哨动画
- 多主色混用
- Hero 区或营销页式包装

## 预览卡片

- 最终导出尺寸固定：`1080 × 1440`
- 预览可缩放，但必须保持严格等比
- 卡片背景保持干净，不加纹理、插画或装饰背景

## 一句话定义

这是一个**以内容生产和结果预览为核心的浅色专业工具界面**。

## 知识卡片渲染规范（局部例外）

> 本节仅适用于 `CoverCardView`、`BulletCardView`、`ExportCardView` 及其所在的右侧预览面板 (`RightPanel`)。
> 工作台（左栏/中栏/顶部）的视觉规范不变，仍遵循上方的浅色专业工具台定义。
> "禁止项"章节中的限制**不适用于卡片渲染子系统**。

### 卡片风格定位

知识卡片采用**暗黑科技风**，参考 Aceternity 风格但保持克制。
定位：高级 AI / Developer Tool 中的知识卡片系统，70% 冷峻高级 + 20% 科技感发光 + 10% 炫酷点缀。

### 卡片颜色体系

| 用途 | 值 |
|---|---|
| Cover 卡背景 | `#0A0F1C` |
| Bullet 卡背景 | `#0D1117` |
| 主标题文字 | `#F0F6FF` |
| 副标题 / Subtitle | `#94A3B8` |
| Bullet 正文 | `#CBD5E1` |
| Bullet 标题 | `#E2E8F0` |
| Tag 标签 | `#60A5FA` |
| 主色（边框/光晕） | `#3B82F6` |
| 页码 | `#2D3F5A` |

### Cover 卡规范

- 背景：`#0A0F1C` + 48px 微网格纹（透明度 3%）
- 顶部：2px 蓝色渐变光边（居中高亮）
- 左上：径向光晕，`rgba(59,130,246,0.18)`，不超出卡片
- Tag：等宽字体，全大写，字间距 `0.22em`，蓝色
- 标题：84px，font-weight 800，`letter-spacing -0.025em`
- 分隔线：72px 宽，蓝色，带轻微 glow shadow
- 副标题：灰色，line-height 1.7

### Bullet 卡规范

- 背景：`#0D1117`，无网格
- 顶部：1px 蓝色微边（透明度 28%）
- 标题：52px，font-weight 700
- 每条 bullet：`borderLeft: 3px solid #3B82F6` + 背景 `rgba(255,255,255,0.025)` + 右侧圆角 6px + 左侧 glow shadow
- 文字大小自适应总字数：<180 字用 30px，<280 字用 28px，≥280 字用 26px

### 预览容器

- `CardPreview` 包裹层：`border-radius: 8px`，`box-shadow: 0 0 0 1px rgba(59,130,246,0.14), 0 8px 32px rgba(0,0,0,0.55)`
- 右侧预览面板背景：`#080C16`（深于卡片背景，形成衬托）

### 导出一致性

- `ExportCardView` 直接复用 `CoverCardView` / `BulletCardView`，无独立样式
- 预览与导出必须视觉完全一致，禁止为导出态单独修改样式

### 禁止（卡片子系统）

- 不做成赛博朋克霓虹海报
- 不做重动画或持续循环动效
- 不引入外部字体（等宽字体使用系统字体栈兜底）
- 不为了炫酷牺牲正文可读性
