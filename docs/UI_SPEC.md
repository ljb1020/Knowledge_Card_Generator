# UI_SPEC.md

## 目标

UI 风格定为：**浅色、专业、克制、结构化的桌面工具台** 与 **硬核的“赛博终端风 (Cyber Terminal)”预览卡片**相辅相成。
不做官网、作品集营销页或普通的展示页面。工作台侧重生产效率与现代高质感，卡片侧重技术极客视觉表现力。

## 布局

主页面固定三栏：

- 左栏：输入与操作、历史记录列表，宽 `320px` 最小宽度。
- 中栏：结构化内容编辑器（支持动态 AutoResize 与按键生成）。
- 右栏：实时预览缩放引擎。

另有导出页：

- `/export/:jobId`，作为 Playwright 无头浏览器的挂载点。

## 整体视觉：Apple 毛玻璃现代工作台

工作台主体（全局、左栏、中栏）：
- **背景**：采用多重径向高光模糊（苹果深色透明虚化感），基于极浅 `F4F5F7` 加上四角的浅蓝、粉白辐射渐变。
- **面板**：`glass-panel`，带高亮边缘倒角的苹果级毛玻璃（`backdrop-filter` 为 48px blur 与 180% 饱和度）。
- **字体族**：主文本使用系统高级字体栈 (Inter + Apple 特色紧组合)。
- **交互**：所有按钮及状态均有细腻的动画或过渡（Hover + Active Scale）。
- 历史记录呈现清晰的状态点（Pulse Dot 闪烁）表示任务是在“生成中”、“导出中”及新加入的“已发布”等状态。

## 卡片面板编辑器

左侧导航与中间编辑器，抛弃厚重的传统样式：
- 采用极为精简现代的自动尺寸适应型 (`AutoResizeTextarea`) 多行文本框。
- Focus 时附带 `ring` 发光过渡 (`#007AFF`)，常态下隐藏无用边框。

## 禁止项（针对工作台）

- 花哨的营销弹窗动画
- 杂乱无章的阴影堆叠（只认系统级柔和倒角）
- 违和的多主题色混搭

## 知识卡片子系统规范（赛博终端风 Cyber Terminal）

> 本节仅适用于 `CoverCardView`、`BulletCardView`、`ExportCardView` 及其所在的右侧预览大屏面板。
> “禁止项”章节不适用。

### 卡片风格定位

系统完全推翻原先的纯暗黑或常规 Aceternity，转向了**技术极客赛博终端风 (Cyber Terminal Hacker Style)**。
定位：70% 系统日志极客风 + 20% 高级科技光带 + 10% 终端绿高亮。

### 导出尺寸约束

- **基础物理尺寸**：固定为 `1080 × 1440` 逻辑宽长。
- **响应展开**：长文自动分两张 `1080 × 1440` 进行连贯播放（即：原设定虽为 4 卡，但在长短文混杂下，渲染层会自动将 Bullet 数 ≥ 4 的物理拆页以符合美观排版）。

### 卡片颜色体系

| 用途 | 值 |
|---|---|
| 统一终端背景 | `#030712` (深幽极黑) |
| 科幻微网格纹 | `rgba(0,240,255,0.04)` |
| 顶部霓虹主发光 | `linear-gradient` / `#00F0FF` (Cyan 蓝) |
| 终端系统特殊字符 | `#00FF41` (Terminal Green 绿) |
| 主大标题与显式文字 | `#FFFFFF` 或 `#E0F4FF` |
| 面板背景缓冲色 | `rgba(0,240,255,0.02)` |

### Cover 卡独有设计

- **光电装饰**：四个角的定位系统 `CornerMark`（准星十字边框）。
- **右侧侧边栏**：悬浮数据流字样（竖向，`JetBrains Mono` 字体，如：`SYS.READY // 0x00FA...`）。
- **主体内容**：超大字号 (`112px`, letter-spacing `-0.05em`)，辅之发光蓝色分离边框。
- **导航（Bento 网格组）**：下半区带有表示进度的发光仪表盘（带有当前进行节点的高亮进度条方块 `ProgressSquares` 及终端标号）。

### Bullet 卡独有设计

- 标题区与封面进度系统布局保持一贯呼应（同样具有右上角系统标志与光晕）。
- **每一行要点首部**：含有 `>_ XX` 形式的终端模拟输入符合（序号递增，`#00FF41` 着色）。
- **内容容器**：文本区域 `rgba(0,240,255,0.018)` 底色结合浅蓝线边框，左侧边缘粗带光晕。
- **智能追问排版**：如果有显式的答或问号（“？”，“答题方向：”等特征匹配），UI 引擎自动渲染高亮淡蓝题干和暗蓝冷色补充说明，打破原本文本块的死板。

## 最新加入：一键分发小红书交互

为服务“发布能力”，在核心工作台处（左面板）：
- `LeftPanel` 左上方存在“测试登录”快捷键 (`/api/xhs/check-auth`)。
- 当渲染记录的状态抵达 `done`，工作台会多显示一个红色主题重点按钮：“小红书”，启动全自动化 Playwright 背景推送程序，发布状态变色成最终的 `已发布 / #published`。
